Attached is another rotation themed cube button, simpler in design than my Rubik's cube button. This cube consists of 8 smaller cubes, with each face dividing into 4 squares, evoking the Windows 8 logo. Hovering over the button will cause the 8 smaller cubes to float apart slightly, and the entire assembly to rotate backwards by 90 degrees, whilst initiating a colour tranisition from light blue to hot pink. Pressing the button will then change the cube to orange. The image contains 11 frames, 9 of which constitute the hover-state animation, the first and last frame being designated as the default and pressed-state frame respectively.

I personally recommend setting the button size in Classic Shell to about 40. If anyone would like a modified colour palette or the animation speed increased/decreased, feel free to drop me a line.

Thanks ZoNi, glad you like it

As Jcee said, animation between hover and pressed state frames isn't currently supported. That said, I'd be more than happy to upload some other versions of this button with different pressed state frames. Any modified versions would probably just feature different colours though, rather than alterations in size and shape. The reason being I would like to avoid drastically altering the size and shape of the final (pressed state) frame because there will be no transition at all between it and the end of the default to hover animation sequence. In other words, the less different the pressed state frame looks, the less jarring the transition from hover state to pressed state will be, if you catch my drift.
