The start menu scans the Start Menu folders and looks for shortcuts. Then it tries to resolve the targets of those shortcuts. This information is needed for features like "frequently used programs" or "highlight new programs". Some of that information is cached, however on startup it has to go and verify if it is up to date (it is faster than the original scan, but not free).

The start menu does not index the files for search purposes. That's done by the indexing service. Look at your indexing settings to see if some unwanted folders are listed there.

See what process is consuming CPU and I/O bandwidth. Also see if not starting the start menu has any effect (you can disable it per user from the General Behavior tab).
