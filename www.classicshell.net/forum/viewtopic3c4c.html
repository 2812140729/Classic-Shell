ASUS K20 Evolution (Win 7 style) Only works with CSM v 4.2.7

The menu skin started out from an Asus K20 minitower, web picture.
Took a while to denoise the jpg compression artifacts.
Then the creative juices started to happen.

It evolved, using many of the new features of the CSM "Metallic" .skin7

The "Asus" logo can be swapped out with a different symbol with transparent background image.
As long as matches same height 70 pixels. Anything bigger will extend on to rest of menu.


The glass option will give a flatter look, with not as much gradient.
For myself, if glass is not overridden in CSM settings to a dark menu color,
I tend to turn off the 2nd column border (DVD drawer).

There is an option for custom image for main menu.
If image is small, tiling repeats the image to fill the menu.
If too large, only the top corner of image is visible. It is mostly to use repeatable texture.
One can put in a picture image.
Would have to edit in a photo editor first, and adjust brightness,
blur, etc, for good menu text visibility.
To make image wrap somewhat around the sides, the image is only 80 percent intensity mixed with the grey background.
It is not going to jump out in full color. Rather subdued.
Once again, best not to use the dvd drawer option with a bright picture image, as its border will look fuzzy.


There is no transparency/blur in this skin.

Oddity note. If you change to have normal black text with black selectors, text will not be visible on split selector.

It appears I might be the first person to use a modified "Metallic" skin.
There could be errors in the way I composed the text script. Don't know everything. I don't see any big errors at this point.
I would have liked to have some of the options grayed out, when they don't apply. I could not figure it out. Sorry.
The file is honking big. This skin is only compatible with the beta CSM version 4.2.7



Feels like back in the 1980s, with brushed metal stereos.

Hey Weboh. Long time no see.

Ivo said it in text somewhere, that the image starts in the top left corner of menu,
top right if language goes right to left.
Because the Metallic , and this skin do not stretch the image like on regular menu skins,
there are good points and bad points. No stretching distortion,
but a big photo image needs to be correct for the users monitor resolution. Small tile image, no problem.

You would need to put in a photo program to resize.
Probably reduce to 540 pixels wide, so the jump view gets most of image, when using standard 96 dot per inch monitor.
Those little stars will only show if there are lots of entries in first column (tall menu).

For HIDPI, image would need to be bigger, or it will run out of image pixels, and tile.
The Metallic skin will show 100% full color image. The same image on K20, mixes with the background grey menu.
So image might need contrast boosted a bit, to make colors vibrant again.
Another thing with the K20 skin is if the top border is added, part of image will be hiding behind the top.
Does really become exacting sometimes.
