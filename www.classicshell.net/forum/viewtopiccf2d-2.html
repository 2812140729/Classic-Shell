Good day.

Ever since the "Minimum Menu Height" setting was added to Classic Start Menu I have been having a hard time understanding it.
The help bubble says it is the minimum number of search results to show.

I recently cleared my UserAssist, so now "Minimum Menu Height" is again relevant until I run 15 programs from the Start Menu. I set my Minimum at 18, which should roughly make my menu the same height as when I have 15 Frequent Programs and 3 pinned programs. Anyway, with Minimum=18 I do a search, and the result is NOT 18 results, it is more like 15 results if we include the category headers, and 16 if we include the extra "search for more" at the bottom. Neither of which is 18. I could increase the Minimum value until I do see 18 results, but that is not the point; the point is: I do not understand this value; it does not seem to be the minimum number of search results :-(

Don't overthink it It is what it is, the menu height. Consider the value as just a number which increases the height. The search results that fit in it *will vary* depending on the small/large icon size, and how many search providers you have added. On my PC, the minimum menu height is 20 but I have 8 search providers defined and icon size of 40 so it can show only 6 search results excluding the category headers.
