Post subject:  Allow jumplists without keyboard shortcuts
Occasionally, I'll open the start menu, move my mouse slightly out of habit, and start typing my search query, only to discover that my mouse has moved over a jumplist arrow and caused my keyboard presses to open some random program.

I like having jumplists on my start menu, but I don't like how typing a key can cause a program to open. I only want programs to open if I click on them or press enter.

Under the "Keyboard shortcuts for programs with jumplists" option, I see three options, but none of them allow you to disable the keyboard shortcuts for jumplists. Please add a fourth option that allows you to do so.
Author:  Ivo [ Sun Dec 11, 2016 8:39 pm ]
Post subject:  Re: Allow jumplists without keyboard shortcuts
Look for the setting "Delay multiplier for split buttons". Set it to some rather large number, like 10000. This will make the delay 100 longer than the normal hover delay, so about half minute. It will effectively disable the hover behavior.
Author:  SingleIssuePoster [ Sun Dec 11, 2016 8:55 pm ]
Post subject:  Re: Allow jumplists without keyboard shortcuts
That mostly fixes it, but it doesn't help if my cursor moves over the little arrow. I have zero desire to ever open a program with the letter keys, so it would still be nice to disable the feature.
Author:  Ivo [ Sun Dec 11, 2016 9:02 pm ]
Post subject:  Re: Allow jumplists without keyboard shortcuts
I'm not sure what little arrow you mean. If it is the arrow for the jumplists, the setting I mentioned will work. If it is the arrow for sub-menus (not jumplists), then increase the general "mouse delay" setting.
