Post subject:  Re: Windows 10 start menu custom background picture
It sounds like blur isnt working on your system for whatever reason: you can remove transparency completely from the image

alternatively you can mess around with this line of code at the end of resource 1 (be sure to compile afterwards)
Main_opacity=Region
This will override the default setting (fullglass), and the option 'increase transparency' as it is defined later in the file.

also you may look into also setting to Region:
Main_opacity2=
Submenu_opacity=

Main_opacity can be solid, region, alpha, glass, fullalpha or fullglass. Solid means the menu will be filled with the Main_background color and the bitmap will be drawn on top. Region means the pixels with alpha=0 will be transparent, and the rest will be opaque. Alpha means that the bitmap will be alpha-blended with the desktop behind it. Glass means the pixels with alpha=0 will be transparent, all the rest will blend between the glass color and the pixel color (alpha=1 is full glass, alpha=255 is fully opaque). Fullalpha and fullglass are the same as alpha/glass but inform the start menu that the background behind the menu items can be transparent.

The blue text is ripped from the help webpage: http://www.classicshell.net/tutorials/skintutorial.html so a quick ctrl+f will help you find that section

I am only guessing, like Jcee, 9 times out of 10, problems are usually because the main bitmap used is 24 bit, instead of 32 bit.
Use a different photo editor that saves/exports as 32 bit bitmap. 8Red + 8Green + 8Blue + 8Alpha =32
If you want transparent bitmap, alter the alpha value 255 to lower, say 200.
Hope this helps.
