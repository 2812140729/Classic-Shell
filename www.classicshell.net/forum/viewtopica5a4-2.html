or is it full alpha transparency at zero, in second column?

Taskbar blur works differently than menu blur.

If you want to make second column with blur (fullglass), even without a glass mask, glass color will show at a low level.
Turning csm Glass opacity setting to 0 can help reduce glass color further.
It would require modification of 32 bit menu bitmap to make your desired rgb brightness/alpha transparency
Could start out with your grey menu color rgb at 45 up to white 255/ transparency could try middle of the road 128 alpha


It would require this text below as well:

Main_opacity=glass
Main2_opacity=fullglass



If you made the scrollbar already - great.
If not, it can probably be done, but the width will automatically stretch to make regular size scroll size.
You could make the stretching part as dark menu color on left of white area.
It would end up with dead space on left between text and white skinny thumb bar.

I could be wrong on a lot of what I just said.

Ok. I just see you have picked another skin to modify more easily.
In that skin, instead of the bitmaps having different alpha value, it's the bitmap mask.
The bitmap masks would need to go to an image editor to adjust alpha values, and put back again.
Looks like at standard 191 alpha.
Hey, maybe black text is better on that menu?

This is where it starts to get more complicated. To have transparency, many bitmaps need to be 32 bit alpha bitmaps.
Like selectors, program tree background changed to different alpha/color text value.
The reason the submenu selector looks very see through, is because it is not a real 32 bit bitmap.
It is only 24 bit with that text color.
If you want semi transparent scroll bar, most of those parts too, and the search bitmap box.
Probably a few more that I forgot.

Hi crvst

I decided to abandon what I made so far, because if what you really want is near white,
then the best skin is Ivo's Windows 8 skin7.
It is stuck on transparency 181/255. What I did was make everything same color,
and put in your thin scrollbar. That scrollbar created speedbumps, hurdles, and moats filled with sewage.
I had to put an option in case someone wants to use skin without menu glass, like on Windows 8.
Make sure to set button correctly, or you will see scrollbar's unmentionables.

Settings on Windows 10
So either put CSM "Glass opacity" setting at 50, or 0. In my opinion 0 is better.
Turn on "Overide glass color" option, and set up a white color like 220

If you are on Windows 7 there are other menu glass settings to use.

If you want to edit the skin bitmaps to lower transparency, everything has to match,
like scrollbar parts, and programs background. The thin scroll bar creates the biggest challenges.

Anyway I think I'm done. It's not going to look like the picture above.
Will look more frosty foggy. I still have the old partial finished skins,
if somebody wants to continue further with it. I put them in zip.

Notes
I don't have any knowledge of how to make area on left side of scrollbar thinner.
I left the borders the way they are. Is not that hard to edit the bitmaps and selectors to make flat.
