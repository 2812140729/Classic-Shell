Yup, with the same technique, it can be done, by moving the user picture so it's visible again,
and adjusting the name data.
It's just that the user picture is referenced to the top left corner of the dragon frame,
so it floats all over the place.
When the column width is narrow, first the user picture gets cut off, and in the extreme,
puppy dragons head get lopped off as well. Not that I'm into decapitation, it looked funny at the time.

If anyone has time to play with it,
they can put this text in the bottom of text script instead:

[USER_IMAGE]
User_image_size=40
User_bitmap=30
User_frame_position=left,15
User_image_offset=305,95
Main_opacity=fullalpha
Main_opacity2=fullalpha


[USER_NAME]
User_font="Segoe UI",bold,22
User_text_color=#FFFFFF
User_glow_color=#000000
User_glow_size=3
User_name_position=25,50,-25,222
;User_name_align=right1
User_name_align=center1

They still have to replace at least the bitmap 30 and possibly menu bitmap 31.
Main menu width adjustment in CSM settings, plays a role in line up of name, and picture,
as well as second column icons on or off.
