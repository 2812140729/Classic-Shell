I believe I've found a bug in the Classic Menu app. I'm running a Surface Pro 4 with Windows 10. In the Classic Menu options, I've got the following setting enabled which I think is the source of the issue:


Basic Settings
Left Click opens: Start Menu in Desktop and Start Screen in Metro


The bug can be created as follows:


1. Enter tablet mode
2. On the Metro Start Screen, tap the Windows icon in the lower lefthand corner. Nothing happens, obviously, as I'm already on the Metro Start Screen. But...
3. Swipe from the left of the screen to open the Action Center.
4. Disable Table Mode


The system then switches over to Desktop mode, but the interesting thing is that the Windows start menu stays open and in Windows 10 Metro mode, not the Classic Shell mode. I'd rather not the menu open when I switch to desktop mode, and I'd also rather it not be in Metro mode either. If I close this start menu and reopen it, it reopens in Classic Shell mode....strange indeed.


I can take screenshots or make some attempt at recording the whole thing if you all need clarification.

Post subject:  Re: Metro Menu opens when switching from Tablet to Desktop
Tablet mode is controlled by the operating system, so while switching between Tablet mode and Desktop mode, some inconsistency between which menu opens is to be expected.

Just press the Win key once to close the wrong menu after toggling Tablet mode.
