Each of Classic Shell's components do check automatically if "Automatically check for new versions" option is on in Classic Shell Update. The Check Now button is an additional way to manually check. Make sure the Windows Firewall or any additional firewall present in your security suite/anti-malware isn't blocking C:\Program Files\Classic Shell\ClassicShellUpdate.exe from connecting to the internet.

This is a serious problem and I need your help troubleshooting it. If you are currently having a problem where the check for new version is failing, please do the following:

  1. Download the attached file (note: it is only for Classic Shell 4.0.6. Later versions include the fix already). Unpack it. It contains a modified version of the ClassicShellUpdate utility with some added logging
  2. Run the tool and click on the "Check now" button. Confirm that it reports an error
  3. Go to the folder %APPDATA%\ClassicShell and look for a file named UpdateLog.txt. Attach the file here
  4. Go to the folder %ALLUSERSPROFILE%\ClassicShell and look for a file named update.ver.err. If it exists, zip it and attach it here

Also please include what version of Windows are you using - 7, 8, 8.1, 32-bit or 64-bit, server or not.
