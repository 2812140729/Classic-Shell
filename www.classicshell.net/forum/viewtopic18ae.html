Post subject:  Interface issue/Feature request: shortcut redirection
As you may know, some programs like uTorrent like to add themselves in the root of "Start Menu" folder, and not in the "Programs" directory. This is rare, but it kinda corrupts Classic Shell's interface (any version). I will show screenshots.

Here you can see a perfectly configured Start Menu:

http://i43.tinypic.com/54acdc.jpg

And here you can see what happens when any program adds a shortcut to the root of "Start Menu" folder:

http://i43.tinypic.com/2r3kos8.jpg

It causes Start Menu to stretch (I highlighted the problem with red marker). The more shortcuts get added, the more stretched Start Menu becomes. Yes, you can simply move shortcuts into "Programs" folder manually, but it would still be nice to have this interface problem corrected, because by default, it will stretch on and on forever.

The only suggestion that comes to mind would be to remove the upper category from start menu, and make Classic Shell show its shortcuts in "Programs" menu instead, like this:

http://i40.tinypic.com/28u22l4.jpg

In fact, Windows XP did just that.

It doesn't have to be this way by default, but it would be very nice to have that option to avoid that dreaded Start Menu stretching.
Author:  Ivo [ Mon Jul 29, 2013 3:13 pm ]
Post subject:  Re: Interface issue/Feature request: shortcut redirection  
Since you are using the beta version, you can switch from using the Start Menu folder to using the Pinned folder. Look for Pinned Programs folder setting in the Main Menu tab. The Pinned folder is the default for the Windwos 7 style.
