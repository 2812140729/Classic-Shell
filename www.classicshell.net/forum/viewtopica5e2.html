By "screen goes blank", I guess that the taskbar disappears as well and reappears? It can be that Explorer.exe is crashing as soon as you click the button to open the Start Menu.

You could try a fix that was recently developed after the release of Classic Shell 4.2.3 to address a crashing issue on Windows 10. But if it does not work in your case, then additional troubleshooting will be required.

Download this DLL file: download/file.php?id=2313 (note that this is 64-bit only and is only if you have version 4.2.3 installed, not for 32-bit)

Then, you need to exit the start menu (right-click on the start button, then Exit)
Put the ClassicStartMenuDLL.dll file from the zip into the C:\Program Files\Classic Shell folder
Run ClassicStartMenu.exe.

See if the crashing stops. If not, then you can try to create a crash dump using these steps: viewtopic.php?f=12&t=6 Analyzing the crash dump will allow the crash to be fixed.

The next version will incorporate this fix.
