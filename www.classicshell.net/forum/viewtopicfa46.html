Author:  Ivo [ Wed Jun 19, 2013 11:09 pm ]
Post subject:  Re: New name for Classic in two columns?
"Classic with two columns" is closely related to the single-column style. Everything is a cascading menu. It uses the same skins. Both styles share 99% of the settings and features. So I think it fully deserves the "classic" name.

The Windows 7 style is an entirely different beast. It has multiple modes (programs, jump lists, search), has variable size depending on the mode, has a completely separate set of skins, and has many unique settings.

The main menu only looks similar to the Windows 7 menu and that is because of the skin. The underlying style of operation/usability is similar to the Windows XP Start Menu where "All Programs" expands like a cascading menu. Earlier it was called the "Windows XP style" but with XP style, people will expect the menu to look like XP too which is a total mismatch for Windows 7/8 look. Hence the name "Classic with two columns". Is perfect IMHO.

Author:  Gaurav [ Thu Jun 20, 2013 1:25 am ]
Post subject:  Re: New name for Classic in two columns?
It is to aid discoverability and prevent users from having to hunt around in settings, making it easier by preconfiguring menu items so less steps required for users. Note that the menu items are different in single column vs two columns. Many users never even change the defaults or can't figure out how to add/remove custom menu items.

The 3 styles are variants of the menu seen in Windows 9x/2000, Windows XP and Windows Vista/7.
