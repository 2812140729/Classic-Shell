Sounds like a bug. Some changes were made to make the drag/drop more compatible with Windows 8. This may have broken the right-click dragging.

Until this is fixed, hold down Shift while dragging to move. The mouse cursor shows what operation will happen. If you see a small [+] in the corner, it will copy. If you see a small arrow, it will create a shortcut. Otherwise it will move. Keep in mind that creating a shortcut to a shortcut actually copies it.

The rules are:
1) If you move stuff within the Programs menu, the default operation is MOVE
2) If you move stuff from the Programs menu to the main menu, the default operation is SHORTCUT, which in case you are dragging a shortcut is the same as COPY
3) At any time you can hold down Shift to force MOVE, hold down Ctrl to force COPY, or both for SHORTCUT

The difference between 1 and 2 was done few versions ago to mimic the Windows 7 menu's Pin behavior. When you pin something it doesn't disappear from the original location. Many people were messing up their start menu by trying to pin programs and losing the original shortcut.

I am very surprised the Shift key doesn't work for you. I tried moving something from Programs\Adobe to the main menu while holding Shift, and it moved just fine.

The mouse cursor should indicate what operation is going to happen - MOVE, COPY or SHORTCUT. Does change change when you press Shift or Ctrl as you are dragging?
