Post subject:  All's Well on Windows 8 and 8.1
I've got the beta on both Windows 8.0 and 8.1 test systems, and it seems to work just fine. I haven't found any bugs.

On Windows 8.0 I have the Search Indexer disabled, which is a bit non-standard, but with my fast SSD-based system the searches are still virtually instantaneous. Good job!
By the way, I use the Taskbar on top of the screen. As always, ClassicShell handles that quite well.

I did think of a feature that should be considered as an improvement though...

On Windows 8 the system applications such as File Explorer and WordPad that have tabbed interfaces do not put a light glow behind the text that they paint in the title bar. That isn't that big a deal with the typical pre-fab desktop colors, which are generally light, but if one chooses one of the non-Automatic colors and adjusts the Brightness down you can get a situation where there's just a dark title bar and the text becomes essentially invisible.

Since ClassicShell adds hooks to File Explorer, would it be possible to hook Explorer.exe in such a way as to paint a background glow behind the title bar text? What I'm asking for is something along the lines of what's done in Internet Explorer, though in the case of File Explorer the application paints the text.

I realize this works around a basic deficiency in the Microsoft software - something that crops up more and more often lately as Microsoft tries to de-emphasize the desktop. But that's precisely the role I see ClassicShell playing more and more - making Windows 8 livable.

-Noel
