Detailed change log: (see here for a blog article with screenshots)

New features in Classic Shell 4.2.5:

● Classic Start Menu's cascading menus are now more powerful than ever. You can now enter two folder paths in the Link property of an item, separated by a semicolon. You will get their combined contents in one sub-menu!
● Support for Search Providers in Windows 7 style also: If a custom item has %1 or %2 in its command, it is considered a Search Provider and will show up during searches at the bottom of the results. You can use this functionality to add any website, or an external program with the appropriate parameters so the Start Menu search box contents will be passed on to this search provider.
● The search for programs and settings reuses the result from the previous search until the new results are calculated for extremely fast searching
● New feature to uninstall apps for Windows 8/8.1 and Windows 10 by right-clicking
● New setting for the Win key and Start button click to open the Windows Menu when in Tablet mode but Classic Start Menu otherwise
● The Metro and Midnight skins support transparency in both columns in Windows 7 style
● New support for per-monitor DPI scaling. The text and menu elements are scaled independently for each monitor. The icon size depends on the global system DPI
● The limit for recent/frequent apps is raised to 40.

Other minor changes:

● When dropping a file on a program in the main menu, the program is highlighted
● The Metro skin uses an accent color so the menu background will change correctly when the wallpaper changes
● The selection in the transparent Metro skins has a border to make it more visible
● Fix for the bug where All Programs doesn't work with the tab key when the touch keyboard is shown
● Fix for the gap that appears in the Classic style when the MRU list is cleared
● When the menu hover time is set to 0, the multiplier for All Programs delay uses the value of 100 instead
● Pressing Enter during the search will execute the first found result when it is available
● Searching the Internet works with Microsoft Edge
● When Edge is the default browser, Favorites shows its bookmarks
● Fix for jumplists not showing up in the new builds of Windows 10
● If a Metro app name fails to resolve, it is added to a blacklist so the menu doesn't become unresponsive while dealing with that app. If the app is later found in the Apps folder, it is removed from the blacklist
● New way of getting the Metro app icons for Windows 8/8.1 and Windows 10

Known issues in this release:

● RemoteApp sessions crash when using Classic Shell on Windows Server 2012 R2 Remote Desktop Services
● Metro app icons with a white background may not show correctly in some circumstances in Windows 7 style treeview (All Programs showing inside main menu)
